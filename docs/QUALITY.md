# QUALITY.md

## 合并前最低要求
```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
```

## 通用质量标准

### 性能
- 首屏主界面在本地开发环境保持流畅，无明显卡顿。
- 首次回答首字出现时间目标不超过 3 秒。
- 预览区折叠、展开、引用跳转动画不应阻塞输入。

### 可访问性
- 上传区、输入区、反馈面板、目录面板必须可键盘操作。
- 关键按钮具备可读 label。
- 颜色对比满足基础可读性。

### 代码质量
- 默认不使用 `any`。
- 共享类型集中定义，避免重复声明。
- 核心状态转换、聊天拼装和工具函数补单测。

### 用户体验
- 上传、解析、问答、反馈全链路都有完整状态反馈。
- 空状态、失败态和超时态必须可演示。
- 所有主要交互在浏览器截图中可验证。

## 测试策略
- 单元测试：状态转换、mock 问答拼装、输入校验。
- 组件测试：关键 UI 状态切换。
- E2E：上传到问答、引用跳转、反馈提交、预览折叠。

## 验收补充
- 最终报告必须附截图证据。
- 若截图暴露 UI 明显缺陷，不算通过，必须继续修正。
