
目标：
这是我们首次使用这种完全靠AI自驱动的方式进行开发，template_docs目录下的文档模板也是首次被使用，过程中有任何改进的意见，单独写一个"改进意见.md"文档, 文档结构你自己发挥。我们将通过本次demo的开发，验证整套机制的可行性，后续持续更新。

我已经提供了这个demo项目的具体技术栈和需求描述，在docs/refer目录下面。接下来四个主线任务时：
1. 仔细分析"技术栈及项目背景.md"和"需求说明.md"。
2. 根据第一步的分析结果，在template_docs/README.md的指引下，在当前项目中初始化文档结构并填充内容，初始化后的文档作为整个项目后续AI开发的规范，所以很重要。
3. 内容填充好并且和我确认后，开始AI自主开发需求，一直到功能开发完成并且自测通过（需要浏览器截图证明各项功能测试都正常），尽可能让AI完全自主完成全部任务，尽量避免人工干预打断。
4. 整个AI自驱动的过程中，包括和人工交互的过程，时刻记得更新"改进意见.md"文档，这个文档将作为我们demo完成后对整个开发过程审视的参考，很重要。

提个小建议，为了避免上下文很快就被占满，估计你需要分别开启多个子Agent来协助你完成。如果都在一个Agent中，你的注意力会被分散的很厉害。
