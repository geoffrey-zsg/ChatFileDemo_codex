背景说明：
刚才我使用claude sonnet 4.6模型，但是生成的项目质量存在很多问题，我将过程问题都记录下来。你不要犯同样的错误。

问题列表：

1. 拆分子Agent执行任务时，只简单的按照功能拆分后一个Agent领域了一个任务就开始干活了，这种应该不够细和合理吧？不同的Agent可以胜任不同的细节工作，比如专门负责UI的Agent，专门负责数据的Agent等等，供你参考。
2. 自测不充分，甚至偷懒，这一点问题很大，页面访问明显有报错，但是自测竟然全都通过。
3. 总是重复启动本地服务，明明服务一直在运行，还会启动，遇到端口被占用报错，又换了个新端口启动服务。为什么不先检查下本地服务是否已经存在了，如果存在就不要再启动了。
4. 首次开发完，UI效果太丑了，既没有遵循需求说明中的一些UI要求，也没有充分利用现有的skills进行美化设计，而且我反复修错，UI页面效果一直都非常差。
5. 自测时必须要浏览器截图验证，方式不限，一种截图方式不行就换一种方式，比如agent-browser, playwright等仔细查看截图，看是否功能正常，UI美观，不要偷懒省略了这一步。
6. 最终自测完成的验收报告中，必须有产品截图进行佐证，发现不合理就继续改。
