# Skill: <技能名称>

> 用一句话说明这个 Skill 解决什么问题。

## 适用场景

- 什么情况下使用这个 Skill
- 需要满足什么前提条件

## 不适用场景

- 什么情况下不应该使用
- 避免误触发

## 输入

需要读取的文档或资源：
- `docs/ARCHITECTURE.md`
- `docs/FRONTEND.md`
- 其他相关文档

## 执行步骤

1. 第一步做什么
2. 第二步做什么
3. 第三步做什么
4. 验证结果

## 输出

- 代码改动
- 文档更新
- 测试结果

## 注意事项

- 关键约束
- 常见坑位
- 验证要求

---

## 示例：前端页面开发

```markdown
# Skill: frontend-page-development

> 基于设计稿和需求开发前端页面，包含 UI、状态、API 接入和基础测试。

## 适用场景

- 需要开发新的业务页面
- 已有设计稿或明确的 UI 需求
- 接口契约已定义（或可用 Mock）

## 不适用场景

- 只是修复单个样式问题
- 只是调整现有页面的小细节
- 接口契约尚未确定

## 输入

- `docs/FRONTEND.md` - 前端开发规范
- `docs/ARCHITECTURE.md` - 架构分层规则
- `docs/DESIGN.md` - 设计系统规范
- 设计稿或需求文档

## 执行步骤

1. 阅读需求，明确页面功能和交互
2. 确定页面在目录结构中的位置（`src/features/<模块>/pages`）
3. 定义数据类型和 API 接口（优先使用生成的类型）
4. 开发 UI 组件（复用现有组件，必要时新建业务组件）
5. 实现状态管理和数据流
6. 处理 loading、error、empty 三态
7. 补充必要的测试
8. 运行 `pnpm lint && pnpm typecheck && pnpm test`

## 输出

- 页面组件代码
- 相关业务组件（如有新建）
- API 调用封装（如有新增）
- 测试用例（单元测试或 E2E）
- 更新路由配置（如需要）

## 注意事项

- 优先复用 `src/shared/ui` 中的基础组件
- 状态管理遵循 FRONTEND.md 中的规范
- 筛选条件优先放 URL query
- 所有异步操作必须有 loading 和 error 处理
- 新增全局模式时同步更新文档
```
