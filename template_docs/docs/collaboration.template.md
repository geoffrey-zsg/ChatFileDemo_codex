## 3. AI 自主工作规范

### 问题记录机制

**核心原则**：纠错成本低，等待成本高。AI 发现需要人工决策的问题时，记录到 `CHANGELOG.md` 后继续工作，不阻塞流程。

**记录位置**：`CHANGELOG.md` 的 "⚠️ 需要人工决策" 章节

**需要记录的情况**：
- 破坏性变更（影响 API 契约、数据结构、用户行为）
- 需求冲突（产品文档与技术约束冲突）
- 架构疑问（不确定的技术选型、分层调整）
- 规范缺失（规范文档未覆盖的场景）

**记录格式**：参见 `CLAUDE.md` 的"问题记录机制"章节

**处理流程**：
1. AI 发现问题 → 记录到 CHANGELOG
2. AI 继续完成其他工作
3. 人工定期查看 CHANGELOG → 决策
4. 问题解决后，从"需要人工决策"移至对应章节或删除
